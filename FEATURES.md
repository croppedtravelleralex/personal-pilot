# FEATURES.md

`PersonaPilot` 的最终功能文档。

目的：单独回答一个问题：

> 这个项目最终要实现哪些能力？

---

## 1. 产品目标

本项目最终要成为一个：

- 运行在 `Windows 11`
- 面向个人研究与自动化工作流
- 具备桌面化任务执行能力
- 可长期维护和演进
- 文档与工程同步推进
- 以 `Tauri 2 + Vite + React + TypeScript` 为桌面壳
- 以 Rust 核心承接本地任务、状态、存储与浏览器执行

的本地桌面自动化系统。

---

## 2. 最终核心功能

### 2.1 任务生命周期管理

- 创建任务
- 查询任务
- 排队任务
- 执行任务
- 取消任务
- 重试任务
- 成功 / 失败 / 超时状态管理

### 2.2 执行引擎抽象

- fake runner 闭环能力
- real runner adapter
- `lightpanda-io/browser` 集成能力
- runner trait / interface 统一

### 2.3 数据持久化

- SQLite 任务存储
- run history 持久化
- logs 持久化
- artifact 元信息管理
- 后续迁移 / 升级能力预留

### 2.4 本地桌面交互

- 首页状态面板
- 任务列表与详情
- persona / continuity 视图
- 日志与报告页面
- 设置页与本地运行控制

### 2.5 指纹与网络策略

- 浏览器指纹配置与注入
- 指纹模板 / 指纹策略
- 高级指纹场景下的性能开销控制
- 代理池管理
- 代理抓取、清洗、去重、验证
- 代理轮换与失败剔除
- 地区感知代理匹配
- 指纹与代理协同策略

### 2.6 资源与稳定性控制

- 并发控制
- 资源限制
- 磁盘使用监控
- artifact / log 保留、清理、归档
- 写放大控制
- 长期运行稳定性保护

### 2.7 观测与验证

- 结构化日志
- 基础错误分类
- 最小 smoke test
- 执行验证机制
- 阶段汇总与结果追踪

---

## 3. 非目标

当前与近期不追求：

- Electron
- 多窗口架构
- 额外 Node 后端服务
- 公网服务化
- 多租户平台化

---

## 4. 阶段映射

### 当前已具备

- 最小任务闭环
- SQLite 持久化
- 内存任务队列
- fake runner
- runs / logs 查询接口
- API Key 鉴权
- Lightpanda runner 基础适配

### 当前未完成但已进入主线

- 桌面壳搭建
- 统一服务层
- 只读状态 / 任务 / 日志页面
- 更强的控制面与观测面

### 中长期能力

- 指纹系统
- 代理池系统
- 身份画像与一致性系统
- 行为模拟
- 策略引擎
- 实验记录系统
