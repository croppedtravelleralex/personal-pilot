# Stage Summary Maintenance Rules (2026-04-02)

## Purpose

Keep the entry summary useful by preventing it from silently turning back into a long planning document.

## Rules

1. **Keep it short**
   - stage status
   - completed in current stage
   - frozen in current stage
   - reopen rule
   - one short “where are we now?” answer

2. **Do not add roadmap sprawl**
   - no long future-planning bullets
   - no multi-stage strategy dump
   - no speculative backlog expansion

3. **Sync before adding**
   - if STATUS / TODO / PROGRESS disagree, fix them first
   - only then update the entry summary

4. **Treat the summary as control surface, not planning surface**
   - planning belongs in docs
   - the summary only answers current-stage state quickly

## Maintenance decision

> If a new line makes the stage summary longer but not clearer, it should stay out.
