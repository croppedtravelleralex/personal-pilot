# VISION.md

## 为什么要有这个文件

为了让项目在长期推进中不跑偏：

- 不只盯着“当前能写什么”
- 还要始终知道“最终要做成什么样”

---

## 最终效果

`PersonaPilot` 最终应当是一个：

- 运行在 `Windows 11`
- 以桌面应用形态交付
- 面向个人研究与自动化工作流
- 具备稳定任务执行能力
- 可持续演进
- 文档与工程同步推进
- 可被本地 UI 与内部服务层调用

的本地桌面自动化系统。

---

## 最终能力清单

### 必备能力

- 任务创建、调度、执行、取消、重试
- 状态查询与结果回传
- 日志记录与 artifact 管理
- 浏览器指纹控制能力
- 代理池与代理策略能力
- 本地数据目录与设置管理
- 长期运行下的资源与稳定性控制

### 开发阶段能力

- fake runner 闭环
- API 可测试
- SQLite 可追踪
- 模块职责清晰

### 演进阶段能力

- 接入 `lightpanda-io/browser`
- 支持真实浏览器自动化
- 支持更强的可观测性与可解释性

---

## 北极星原则

1. 先把闭环打通，再追求复杂度
2. 先把模块边界写清楚，再扩功能
3. Rust 核心与桌面壳必须职责清晰
4. 每次推进都要留下文档与日志
5. 任何代码推进都要能回答：它离最终本地产品更近了吗？
