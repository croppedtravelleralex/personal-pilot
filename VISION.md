# VISION.md

## 为什么要有这个文件

为了让项目在长周期推进中不跑偏：
- 不只盯着“当前能写什么”
- 还要始终知道“最终要做成什么样”

---

## 最终效果

`AutoOpenBrowser` 最终应当是一个：

- 运行在 Ubuntu 上
- 主要供个人**学习与研发**使用
- 面向高性能自动化任务执行
- 可持续演进
- 文档与工程同步推进
- 可替换执行引擎
- 可被 API 调用与集成

的浏览器自动化系统。

---

## 最终功能清单

### 必备能力
- 任务创建
- 任务调度
- 任务执行
- 任务取消 / 重试 / 超时控制
- 状态查询
- 结果返回
- 日志记录
- artifact 管理
- 浏览器指纹控制能力（高级指纹下仍尽量保持较小性能开销）
- 代理池与代理策略能力
- 持续抓取代理的工具（优先基于开源项目改造）
- 所有访问强制经过代理池
- 代理池按地区维持基础可用存量
- 代理池根据并发动态调节可用阈值与库存策略
- 具备磁盘使用控制、落盘节制与性能优化意识，避免长期运行时把宿主机写爆

### 开发阶段能力
- fake runner 闭环
- API 可测试
- SQLite 可追踪
- 模块职责清晰

### 演进阶段能力
- 接入 `lightpanda-io/browser`
- 支持真实浏览器自动化
- 支持更高稳定性与更强观测能力

---

## 北极星原则

1. 先把闭环打通，再追求复杂度
2. 先把模块边界写清楚，再扩功能
3. fake runner 和 real runner 必须共用抽象层
4. 每次推进要留下文档与日志
5. 任何代码推进都要能回答：它离最终目标更近了吗？
