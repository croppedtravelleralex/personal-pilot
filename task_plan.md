# Task Plan

## Goal
为 `agent.alexstudio.top` 设计并落一版**风险可控、仅供本人测试**的 API 中转站最小方案，重点先把边界、网关职责、鉴权与日志/限流约束收清楚，再决定如何把上游额度链接进来。

- Phase 2 — 最小网关架构设计: pending
- Phase 3 — 本人测试版安全边界与运维约束: pending
- Phase 4 — 落地清单与下一步实施路径: pending

## Key Decisions
- 当前目标不是做“开放网关”，而是做“本人测试可控网关”。
- `agent.alexstudio.top` 应被视为受控 API 入口，而不是直接暴露上游额度凭据的透明代理。
- 优先把鉴权、限流、日志边界说清楚，再谈后端如何接上游额度链。

## Risks
- 如果把上游凭据直接暴露到客户端，请求一旦泄露将直接失控。
- 如果不限制来源、不限速、不做 token 撤销，测试入口会快速变成高风险公开入口。
- 如果日志保留完整 Authorization / Cookie / body，最容易形成高危泄露面。
